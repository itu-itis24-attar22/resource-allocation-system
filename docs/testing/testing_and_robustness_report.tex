\section{System Overview}
The Resource Allocation System is a C++ prototype for processing university space requests. The system loads users from \texttt{users.csv}, spaces from \texttt{spaces.csv}, and requests from \texttt{requests.csv}. It evaluates one-time and recurring requests using rule-based checks and produces two external outputs: \texttt{allocations.csv} for approved allocations and \texttt{request\_results.csv} for processed request outcomes, including lifecycle history.

\section{Testing Goal}
The testing effort was designed to discover failures rather than only confirm expected behavior. Testing included normal requests, malformed rows, boundary values, invalid references, duplicate identifiers, recurring self-conflicts, CSV export edge cases, empty files, and adversarial datasets intended to break hidden assumptions in the implementation.

\section{Bugs Found and Fixed}
The focused hardening pass addressed the following confirmed issues:
\begin{itemize}
    \item Invalid time slots were previously accepted and could produce invalid allocations.
    \item Invalid user and space references were previously skipped instead of being recorded as rejected requests.
    \item Malformed request rows could disappear from reporting or terminate parsing unexpectedly.
    \item Recurring requests could contain duplicate or overlapping internal time slots.
    \item CSV output could be corrupted when field values contained commas.
    \item Duplicate request IDs were accepted and processed normally.
    \item Invalid requests were not shown in main console reporting.
    \item Stale output files could remain after initialization failure.
\end{itemize}

\section{Remaining Limitation}
Malformed \texttt{users.csv} and \texttt{spaces.csv} rows are currently skipped with warnings rather than exported as structured invalid-entity results. For the current design, this is acceptable because the main request-processing robustness requirement is satisfied without introducing a larger reporting subsystem for invalid entities.

\section{Final Verdict}
The system now handles normal, invalid, malformed, boundary, and adversarial inputs correctly. No functional regressions were found after the focused hardening pass.
